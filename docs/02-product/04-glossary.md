# Glossary

**Version** : 1.0

---

# Identity

Représentation numérique d'un acteur pouvant être authentifié.

---

# Login Identifier

Information permettant de retrouver une Identity avant l'authentification.

---

# Credential

Moyen permettant à une Identity de prouver son identité.

---

# Authentication

Processus consistant à vérifier qu'une Identity est bien celle qu'elle prétend être.

---

# Authorization

Processus permettant de déterminer ce qu'une Identity est autorisée à faire.

---

# Security Policy

Objet métier définissant les règles de sécurité du système.

---

# Client

Application utilisant ComLabs IAM pour authentifier ses utilisateurs.

---

# Session

Connexion active entre une Identity et ComLabs IAM.

---

# Access Token

Jeton représentant une authentification réussie.

---

# Refresh Token

Jeton permettant d'obtenir un nouveau Access Token sans nouvelle authentification.

---

# Role

Ensemble de permissions accordées à une Identity.

---

# Permission

Autorisation permettant d'effectuer une action précise.

---

# Domain

Ensemble cohérent de concepts métier partageant la même responsabilité.

---

# Aggregate

Concept DDD représentant une frontière de cohérence métier.

---

# Value Object

Objet défini uniquement par ses valeurs et ne possédant pas d'identité propre.
